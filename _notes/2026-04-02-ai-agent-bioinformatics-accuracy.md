> 💡 Gavin 在 2026-04-02 洗澡时想到的问题（值得深思）

**AI Agent 调用生信工具的准确性问题**：

目前 AI Agent 使用生信命令的准确率很低，核心原因是：

1. **LLM 幻觉严重**：参数、flag、路径、文件格式都会编造
2. **每个工具都要单独写 Skill**：本质是在打补丁，不优雅
3. **软件设计不考虑 Agent**：工具诞生时就没想过会被程序调用

**思考**：与其每个工具都构建一个 Skill（碎片化、不通用），不如从根本解决：

- **软件层**：从设计之初就考虑 Agent 调用（标准化 `--help`、JSON 输出、exit code 一致）
- **工具描述层**：MCP、Tool Schema 这类统一接口标准
- **Agent 层**：能用自然语言准确理解工具行为，而不是靠 Skill 背书

就像程序设计从汇编 → 高级语言，工具调用也应该有个更通用的抽象层。**Skill 只是过渡方案，不是终态。**
