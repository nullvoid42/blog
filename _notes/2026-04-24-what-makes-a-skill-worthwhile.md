## 什么情况下我们需要做一个 Skill

**当 Agent 不会用一个 CLI 时，需要做吗？**

不需要。这是知识，大模型什么都能学。知识不需要装进 Skill。

Skill 的价值不在于"教模型知识"，而在于**强制约束工作流**——删了它，流程会出现明显问题。

## 目前 Skill 市场繁荣，但也十分混乱

五花八门的 Skill 说明生态还在早期摸索阶段。

**这么多 Skill 都有价值吗？**

知识类 Skill 注定被大模型内化，只是时间问题。

**未来会怎么演化？**

- 短期：通用知识类大量淘汰
- 中期：剩下安全审查、个人工作流规范
- 长期：Skill 进化为"什么情况下该调用哪个 Agent / 工具"的更高级抽象

## 个性化的工作流程更需要 Skill

只有**个人定制**的规范才值得固化：

- 你的服务器配置和运维习惯
- 你的写作风格和模板
- 你的私有 API 封装和偏好设置
- 多工具串联的固定顺序（顺序错了会翻车）

**通用知识不值得做 Skill，个人规范才值得。**
